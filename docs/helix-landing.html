The one idea

A puppet became a peer.

Klipper pre-computes an exact stream of step pulses and the micro-controller replays it, tick by tick — the only safe design when MCUs were 8-bit parts with no memory and no clock. The boards on your printer today are 32-bit computers. HELIX changes the one decision underneath everything.

The old bargain

MCU as puppet

  • The host owns the truth of position.
  • The link is a firehose with a deadline.
  • Any hiccup ends in shutdown() — heaters off, motors free, print abandoned.
→
The HELIX bargain

MCU as peer

  • The board owns its clock, position, and queue.
  • Deep intention queues absorb link jitter as slack.
  • A recoverable failure becomes pause and hold — not death.

What changed

Seven shifts, one protocol.

Everything below follows from giving the micro-controller a clock and a memory it can be trusted with.

01 · Motion

Intentions, not step streams

stepper-only pulses → actuator-agnostic segments

A segment says where the joint should be, not which pulses to send — so the actuator is a swappable backend: steppers and PWM/DAC today, a closed-loop BLDC/FOC servo joint tomorrow, on one queue. Up to quintic Bézier, zero drift.

02 · Time

A machine that agrees on the time

host-owned time → disciplined shared clock

Every board disciplines its clock to one machine time, so "do this at T" means the same instant across the whole machine.

03 · Recovery

Pause, hold, resume

shutdown-everything → hold and continue

Finish queued motion, hold position with heaters on their failsafe policy, keep an execution log, and reconcile the exact stopping point on resume.

RFC 0001 · doc 08 ↗
04 · Transport

Networks as first-class links

USB/CAN only → authenticated UDP, FEC, signed

The same HMAC-authenticated datagram protocol over UDP, CAN, USB and UART; a DTLS-class session, forward error correction, and Ed25519-signed images.

RFC 0001 · doc 07 ↗
05 · Triggers

Hardware events — a capability unlock

polling → interrupts, comparators, DMA

µs stops are the surface. Events off the timer list make things polling made impossible in a motion loop: catching an overrun the instant it happens, DMA-driven ADC oversampling, analog window triggers — reachable now, not a fight with the scheduler.

RFC 0001 · doc 09 ↗
06 · Library

One protocol, declared not generated

code generation → annotation registration

A command is a macro beside its handler that registers itself before main(); the data dictionary is a serialization of the live registry, served not scraped.

RFC 0001 · doc 10 ↗
07 · Reach

One firmware across families

per-chip code → a versioned syscall surface

STM32 and ESP32 speak the same protocol and expose the same versioned board syscall table — a module is written once, not once per chip. The ESP32 becomes a network-native target.

What it buys you

A loose cable stops ruining prints.

Before — the toolhead cable works loose
  1. Communication times out.
  2. The machine shuts down.
  3. The bed goes cold; the part cools.
  4. Adhesion releases — the print detaches. Hours lost.
With HELIX — the same loose cable
  1. The board finishes its queued motion and holds.
  2. Heaters stay on their failsafe policy — the bed stays hot.
  3. You reseat the cable; the board re-handshakes.
  4. It reconciles the exact stopping point and resumes.

Heritage

HELIX is a friendly fork of Klipper, created by Kevin O'Connor and its community — years of brilliant engineering without which none of this would exist. It is Free Software under the GNU GPLv3, preserves Klipper's copyrights and attribution throughout the source, and contributes its ideas back in the open. HELIX is an evolution of Klipper, and no longer Klipper itself — a different bargain for a newer generation of hardware.

Honest about maturity: every HELIX capability is opt-in, and each design doc marks what has run on silicon and what still awaits hardware bring-up. The classic Klipper paths remain, untouched, alongside them.
